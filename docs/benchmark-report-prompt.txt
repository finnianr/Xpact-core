

Folder /home/finnian/Dev/Eiffel/library/Xpact-core/benchmarks
has the following folders corresponding to test files:

Legislation.xml
nes96.xml
ns_att_test.xml
recset.xml
vision2.ecf

Each folder is filled with benchmarking results for various benchmark comparisons between Xpact and eXpat.

The first and last entry correspond respectively to
$EIFFEL/library/Xpact-core/examples/data/Legislation.xml
$ISE_LIBRARY/library/vision2/vision2.ecf

The remaining files are found in $HOME/Dev/C/libexpat/testdata/largefiles

The results in each folder look like this:

'Xpact VS eXpat.CRC-32-attribute.log'
'Xpact VS eXpat.CRC-32-cdata.log'
'Xpact VS eXpat.CRC-32-comment.log'
'Xpact VS eXpat.CRC-32-tag.log'
'Xpact VS eXpat.CRC-32-text.log'
'Xpact VS eXpat.tag_count.log'

Each one is a log of many comparative benchmarking sessions for the same programs.

The 'Xpact VS eXpat.tag_count.log' is measuring the Eiffel parser TAG_COUNTER against the equivalent eXpat program: 
Xpact-core/C-examples/tag-counter/xml_tag_counter.c

All it does is count frequency of occurrences of tag names in a document.

The remain logs files are measuring the Eiffel parser CRC_32_GENERATOR against the equivalent eXpat program:

Xpact-core/C-examples/XML-crc_32/src/xml_crc_32.c

This program calculates a CRC-32 value for a data type specified on command line. The different types are represented by this Eiffel table:

feature {NONE} -- Constants

	Parse_event_types: HASH_TABLE [INTEGER, STRING]
		once
			create Result.make_from_iterable_tuples (<<
				[Tok_text, "text"],				-- text content
				[Tok_cdata, "cdata"],			-- CDATA text content
				[Tok_comment, "comment"],		-- comment
				[Tok_tag, "tag"],					-- tag name (open element)
				[Tok_attribute, "attribute"]	-- attribute value
			>>)
		end
		
The benchmark logs are named according to the strings in this table.

The full set of reports are generated by the use of these two bash scripts:
Xpact-core/scripts/benchmark_Xpact_crc_32.sh
Xpact-core/scripts/benchmark_Xpact_tag_count.sh
